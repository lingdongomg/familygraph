# Change: 优化图谱连线渲染 — 隐藏同辈虚线并改进亲子 bracket 连线

## Why
1. **同辈虚线冗余且干扰**：兄弟姐妹之间的灰色虚线与亲子 bracket 连线叠加在一起，影响阅读。兄弟姐妹关系已通过共同的父母 bracket 线隐含表达（同一个横线分出的竖线），无需额外的直线连接。
2. **亲子 bracket 连线不够美观**：当前实现是从每个父母节点分别画一根竖线到 midY 横线，当夫妻两人有共同子女时，会画出两套独立的 bracket 线（父亲一套、母亲一套），视觉上线条交叉杂乱。

## 用户期望的连线样式
用户的描述是："父代两根线先竖着再横着连起来，然后一根竖线向下，竖线后再一根横线，横线下再展开多根竖线连接多个子代。"

解读为标准家谱图连线方式：
```
  父亲          母亲
    |              |
    +------+-------+      ← 夫妻横线（已有配偶红线）
           |
           |               ← 从夫妻中点向下的竖线
           |
    +------+------+        ← 子代分发横线
    |      |      |
   子A    子B    子C       ← 竖线连接到每个子代
```

关键要点：
- 将夫妻视为一个单元，从夫妻中点（而非每个父母分别）向下引出一条竖线
- 竖线到子代分发横线
- 横线分出多条竖线连接到每个子代
- 如果父/母只有一方在图谱中（无配偶），则直接从该节点向下连线

## What Changes
- **移除同辈虚线**：在绘制非亲子边时，跳过所有 SIBLING 类型边，仅保留配偶红线
- **改进亲子 bracket**：将亲子边按"夫妻对"分组（而非按单个父母），从夫妻中点向下画竖线到分发横线，再分出竖线连接子代

## Impact
- Affected specs: graph-rendering
- Affected code: `miniprogram/components/family-graph/index.js:208-330` — 边绘制逻辑
