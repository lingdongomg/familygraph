## 新增需求

### 需求: 加入申请通知
系统应在新的加入申请提交时，向家庭的所有 Owner 和 Member 用户发送微信订阅消息。

#### 场景: 新加入申请通知
- **当** 用户提交家庭加入申请
- **则** 所有已授权订阅消息模板的 Owner 和 Member 用户收到通知，内容包含申请人昵称和其声称的 Person 身份

### 需求: 加入申请结果通知
系统应在加入申请被通过或拒绝时，向申请人发送微信订阅消息。

#### 场景: 通过时通知
- **当** 加入申请被通过
- **则** 申请人收到订阅消息，告知已被接纳加入家庭

#### 场景: 拒绝时通知
- **当** 加入申请被拒绝
- **则** 申请人收到订阅消息，告知申请已被拒绝

### 需求: 订阅消息授权
系统应在适当时机（如创建家庭时、打开审批页面时）请求用户授权订阅消息。拒绝授权的用户仍可正常使用应用，但不会收到推送通知。

#### 场景: 用户拒绝通知授权
- **当** 用户拒绝订阅消息授权
- **则** 应用继续正常运行，不发送推送通知
- **且** 用户仍可在审批页面手动查看加入申请
