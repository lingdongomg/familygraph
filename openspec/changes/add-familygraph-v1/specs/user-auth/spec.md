## 新增需求

### 需求: 微信登录
系统应通过微信小程序登录流程认证用户，从微信服务器获取用户的 openid，并以 AES-256-CBC 加密方式存储到 `users` 集合中。

#### 场景: 首次登录
- **当** 新用户打开小程序并授权登录
- **则** 系统创建一条 `users` 记录，包含加密的 openid、昵称和头像 URL
- **且** 返回用户资料给客户端

#### 场景: 老用户登录
- **当** 已有账号的用户打开小程序
- **则** 系统通过 openid 检索已有的 `users` 记录
- **且** 返回用户资料，不创建重复记录

### 需求: 用户资料管理
系统应允许已认证用户更新其昵称和头像。

#### 场景: 更新资料
- **当** 已认证用户更新昵称或头像
- **则** 系统更新 `users` 记录中的对应值
- **且** 将 `updated_at` 设为当前服务器时间
