# Change: 修复亲子 bracket 连线比例 — 分布线位置居中、子代竖线方向修正

## Why

上一次 `fix-parent-child-bracket-aesthetics` 将 bracket 起点改为父母节点底部+标签区域下方，终点改为子节点顶部边缘，但实际效果有两个明显问题：

1. **亲代到分布线的竖线视觉上太长**：分布线（midY）位于 `originY + 80`（节点中心 + 30 半径 + 30 标签区 + 20 间隔），距离亲代节点中心 80px。而从分布线到子代顶部只有 30px（`originY + 140 - 30 - 80 = 30`）。上长下短，比例失衡。
2. **子代竖线连接位置不直觉**：当前子代竖线终点在 `child.y - nodeRadius`（子代圆形顶部），但由于分布线 midY 和子代圆顶之间只有 30px 间距，横线看起来几乎贴着子代名字标签区域，视觉上像是从子代名字下方连上来的。

### 期望效果

```
  父亲 ──── 母亲          ← 配偶红线
        |                  ← 短竖线（节点底部 + 标签区下方引出）
   ┌────┼────┐             ← 分布横线（居于亲代底部与子代顶部之间的中点）
   |    |    |             ← 适当长度的竖线
  子A  子B  子C            ← 竖线终点在子节点顶部
```

关键：分布横线应处于亲代底部与子代顶部之间的 **居中位置**，使上下两段竖线长度近似相等。

## What Changes

仅修改 `miniprogram/components/family-graph/index.js` 中 bracket 绘制逻辑（约第 294-352 行）：

- **分布线位置**：从固定偏移 `startY + BRACKET_GAP` 改为 **亲代底部与子代顶部之间的中点** `(startY + childTopY) / 2`，确保上下竖线等长
- **移除 BRACKET_GAP 常量**：不再需要固定偏移，改用动态居中计算
- **对单子女、多子女场景统一应用居中分布线**

## Impact
- Affected code: `miniprogram/components/family-graph/index.js` — bracket 绘制部分（~15 行修改）
- 不影响力导向布局、配偶红线、节点渲染、点击命中检测
